# Task 3 – Web Application Security Testing

## Overview
This project is part of my cybersecurity training.  
It demonstrates common web application vulnerabilities using DVWA (Damn Vulnerable Web Application) and security testing tools.

---

## Objectives
- Understand common web vulnerabilities  
- Perform basic exploitation in a controlled lab environment  
- Learn how attackers exploit insecure applications  
- Use tools like Burp Suite for request analysis  

---

## Tools Used
- DVWA (Damn Vulnerable Web Application)  
- Burp Suite Community Edition  
- Browser Developer Tools (Chrome/Firefox)  

---

## Vulnerabilities Tested
- SQL Injection  
- Reflected Cross-Site Scripting (XSS)  
- Stored Cross-Site Scripting (XSS)  
- Cross-Site Request Forgery (CSRF)  
- Local File Inclusion (LFI)  
- HTTP Request Interception using Burp Suite  
- Security Headers Analysis  

---

## Evidence
All screenshots and supporting files are included in the repository folders.

---

## Key Learnings
- Understanding how insecure input handling leads to attacks like SQL Injection and XSS  
- Importance of proper session management and request validation  
- Role of security headers in protecting web applications  
- Basics of intercepting and analyzing HTTP requests using Burp Suite  

---

## Disclaimer
This project was performed in a controlled lab environment (DVWA) for educational purposes only.
